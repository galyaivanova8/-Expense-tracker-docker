# Expense Tracker

## Описание

Този проект представлява уеб приложение за следене на лични разходи. Потребителят може да добавя нови разходи, да ги разглежда и при нужда да ги изтрива.

Проектът е контейнеризиран с Docker и Docker Compose и се състои от две основни части:

* Backend приложение, разработено с Node.js и Express
* MySQL база данни

## Структура на проекта

```text
expense-tracker-docker
│
├── backend
│   ├── Dockerfile
│   ├── package.json
│   └── server.js
│
├── db
│   └── init.sql
│
├── compose.yml
└── README.md
```

## Използвани технологии

* Node.js
* Express
* MySQL
* Docker
* Docker Compose

## Стартиране на проекта

За стартиране на проекта се използва командата:

```bash
docker compose up --build
```

След успешно стартиране приложението може да бъде отворено на адрес:

```text
http://localhost:3000
```

За спиране на контейнерите се използва:

```bash
docker compose down
```

## Как работи приложението

Потребителят въвежда описание на разхода, неговата стойност и избира категория.

След натискане на бутона „Добави“, информацията се записва в базата данни и се показва на страницата.

Освен това приложението изчислява:

* общата сума на разходите;
* средния разход;
* най-големия разход;
* най-често използваната категория.

При грешно въведен разход той може да бъде изтрит чрез бутона „Изтрий“.

## Комуникация между услугите

Приложението и базата данни работят в отделни Docker контейнери.

В Docker Compose е създадена връзка между тях, като backend приложението използва името на услугата `db`, за да се свърже към MySQL базата данни.

По този начин двете услуги могат да комуникират помежду си без значение на коя машина е стартиран проектът.

## Заключение

Чрез този проект се демонстрира използването на Docker и Docker Compose за контейнеризация на приложение с база данни. Проектът показва как няколко услуги могат да работят заедно в отделни контейнери и да бъдат стартирани само с една команда.
